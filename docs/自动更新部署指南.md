# 自动更新部署指南

更新文件托管在阿里云服务器（Apache），用户打开 App 后会自动检查更新，也可在「设置 → 关于应用」手动检查。

`dev_tools/` 目录下提供了两个 Node.js 脚本，Windows 和 macOS 均可直接运行（需要系统已安装 OpenSSH，Windows 10+ 和 macOS 均已内置）。

---

## 一、服务器配置（只需做一次）

在项目根目录执行：

```bash
node dev_tools/server-setup.js
```

脚本会自动完成：
1. SSH 连接服务器（会提示输入密码）
2. 创建 `/var/www/html/biu/updates/` 目录并设置权限
3. 写入 Apache 配置，确保该路径不被代理拦截、作为静态文件直接访问
4. 验证 Apache 配置语法并重载

**执行成功后验证：** 浏览器打开 `http://8.163.0.99/biu/updates/`，显示 **Forbidden / You don't have permission** 表示正常（目录为空所以禁止列目录）；若显示 **Service Unavailable**，说明配置未生效，重新运行脚本即可。

---

## 二、每次发布新版本的步骤

### 第一步：修改版本号

编辑项目根目录的 `package.json`，升级 `version` 字段：

```json
{
  "version": "1.1.0"
}
```

> 版本号必须严格递增，否则 App 不会提示更新。

### 第二步：打包

```bash
pnpm build
```

打包完成后，产物在 `dist/artifacts/` 目录下。

### 第三步：上传到服务器

```bash
# 上传所有平台的文件
node dev_tools/upload-update.js

# 或只上传指定平台
node dev_tools/upload-update.js --win
node dev_tools/upload-update.js --mac
node dev_tools/upload-update.js --linux
```

脚本会自动：
- 读取 `package.json` 中的版本号
- 扫描 `dist/artifacts/` 中实际存在的文件
- 逐个 SCP 上传到服务器（会提示输入密码，配置 SSH 密钥后免密）

上传的文件说明：

| 文件 | 平台 | 说明 |
|------|------|------|
| `Biu-x.x.x-win-setup-x64.exe` | Windows | NSIS 安装包（支持自动更新） |
| `Biu-x.x.x-win-setup-x64.exe.blockmap` | Windows | 增量更新用 |
| `latest.yml` | Windows | **版本元数据（必须上传）** |
| `Biu-x.x.x-mac-arm64.dmg` | macOS arm64 | 安装包 |
| `Biu-x.x.x-mac-x64.dmg` | macOS x64 | 安装包 |
| `latest-mac.yml` | macOS | **版本元数据（必须上传）** |
| `Biu-x.x.x-linux-x64.AppImage` | Linux | AppImage（支持自动更新） |
| `latest-linux.yml` | Linux | **版本元数据（必须上传）** |

## 三、各平台更新行为说明

| 平台 | 安装方式 | 更新行为 |
|------|----------|----------|
| Windows NSIS 安装版 | `win-setup-*.exe` | 自动下载，点击「退出并安装」完成更新 |
| Windows 便携版 | `win-portable-*.exe` | 不支持自动更新，弹出「前往下载」按钮，用浏览器手动下载 |
| macOS | `.dmg` | 不支持自动更新，弹出「前往下载」按钮，用浏览器手动下载 |
| Linux AppImage | `.AppImage` | 自动下载，点击「退出并安装」完成更新 |

## 四、验证更新功能

### 方法一：在 App 内手动触发

1. 打开 App → 进入「设置」→ 找到「关于应用」
2. 点击「检查更新」按钮
3. 如果服务器上的版本比当前版本高，会弹出更新提示弹窗

### 方法二：验证服务器文件可访问

浏览器访问以下地址，确认能看到 YAML 文本内容：

```
http://8.163.0.99/biu/updates/latest.yml        # Windows
http://8.163.0.99/biu/updates/latest-mac.yml    # macOS
http://8.163.0.99/biu/updates/latest-linux.yml  # Linux
```

`latest.yml` 内容示例：

```yaml
version: 1.1.0
files:
  - url: Biu-1.1.0-win-setup-x64.exe
    sha512: xxxxxx==
    size: 85000000
path: Biu-1.1.0-win-setup-x64.exe
sha512: xxxxxx==
releaseDate: '2026-03-13T00:00:00.000Z'
```

### 方法三：开发模式下测试

开发模式下 updater 使用 `electron/updater/dev-app-update.yml`（已配置为同一台服务器）。只要服务器上的 `latest.yml` 版本号高于 `package.json` 中的版本，开发模式下点击「检查更新」也会触发完整更新流程。

## 五、常见问题

**Q：上传了文件但 App 检查不到更新？**
- 确认 `latest.yml` 已上传且版本号高于当前安装版本
- 浏览器访问 `http://8.163.0.99/biu/updates/latest.yml` 确认可以读取
- 检查 Apache 是否允许访问该目录（先运行 `node dev_tools/server-setup.js`）

**Q：点击下载后没有进度或失败？**
- 检查安装包文件（`.exe` / `.AppImage`）是否已上传到服务器
- 确认 `latest.yml` 中的文件名与实际上传的文件名完全一致

**Q：macOS 用户如何更新？**
- macOS 不支持静默自动更新（需要代码签名和公证）
- App 会弹出「前往下载」按钮，用户点击后在浏览器打开下载页，手动下载 `.dmg` 安装

**Q：脚本提示找不到 `ssh` 或 `scp` 命令？**
- Windows：确认系统为 Windows 10 1809 及以上版本（已内置 OpenSSH）；若没有，在「设置 → 应用 → 可选功能」中安装「OpenSSH 客户端」
- macOS：已内置，无需安装
