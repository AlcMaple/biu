## 📖 详细说明

> 完整技术方案见 [`docs/lyrics-alignment-spec.md`](../lyrics-alignment-spec.md)
> （740 行，含 ASR API 规格 / DTW 完整实现 / LRC 生成 / 部署配置）。
> 这里只记关键决策和已经否过的方向，让下次回来 5 分钟内 ramp up。

### 关键决策

**ASR 选 fun-asr**：
- 阿里达摩院最新模型，**明确支持歌唱识别（带 BGM 整首歌曲转写）**——省掉
  客户端做 Demucs 人声分离的所有麻烦
- 字级时间戳精度高，REST 异步任务模型，国内可直访
- paraformer-v2 同家但不专门做歌曲，留作 fallback

**音频直传 OSS，不经服务器**：
- 一首 mp3 5-10MB，走服务器中转白白占带宽 / 磁盘
- 用 STS 临时凭证（15 分钟有效）控制权限，安全可接受
- 服务端只处理小 JSON 请求，1GB 内存绰绰有余

**DTW 代价设计**：
- `INSERT_COST = 0.3`（ASR 多识别字符，罚分轻）<
  `DELETE_COST = 1.2`（真实字符没匹配到，罚分重）——「宁可让 ASR 多识别
  也不漏真实字符」
- `dp[0][j] = 0`：允许 ASR 序列开头任意跳过 → 自动跳过前奏
- 末尾选 `bestJ`：允许 ASR 末尾任意跳过 → 自动跳过尾奏
- 中间间奏 ASR 自然听不到 → `interpolateNulls` 线性插值填补

**输入是 LRC 而非纯文本（含旧时间戳）**：
- 用户拿到的歌词通常已经是 LRC（网易云 / LrcLib / B 站抓的），只是时间
  戳不准；让他们去掉时间戳再喂进来既麻烦也丢信息
- `parseLrc` 拆出「metadata / lyric」，元数据行（`[ar:]` `[ti:]`、纯时间戳
  间奏标记、空行）按 `raw` 字段原样透传到输出
- 旧时间戳作软先验：DTW 匹配代价加一项 `|priorTime - asrTime| / 1000 * priorCostPerSec`，
  默认 `0.05/秒`（1 秒偏差只多 0.05 分，远小于字符不匹配的 1 分）
- 调参经验：`0` = 完全忽略旧时间戳；`0.05` = 默认；`0.2 ~ 0.5` = 旧时间戳
  基本可信只做小修正

**按行而非按字符回填时间戳**：
- 行级 LRC 是绝大多数播放器（包括 Biu）唯一支持的格式
- 每行取「第一个非标点字符的修正时间戳」作为行时间戳，避免行内演唱节奏
  抖动被错误放大成行间偏移
- 字符级 LRC（卡拉 OK 模式）见 spec 10.1，是可选增强，当前阶段不做

### 已经否过的方向

- **客户端 Demucs 人声分离**：fun-asr 已支持带 BGM 整曲识别，Demucs
  完全多余，包体也不可接受
- **「传歌词 + 音频直接返回 LRC」的现成 API**：国内云没有这种接口，
  必须自己组合 ASR + DTW
- **OpenAI Whisper API**：需魔法，pass
- **任务瞬时完成假设**：fun-asr 是异步任务，必须轮询 `task_status`
  （PENDING / RUNNING / SUCCEEDED / FAILED），建议 2-3s 间隔，5 分钟超时
- **API Key 下发客户端**：`DASHSCOPE_API_KEY` 主账号只在服务端，客户端
  OSS 上传只用 STS 临时凭证
- **副歌自动展开**：歌词只写一遍但实际唱 N 遍时，DTW 只会对齐第一遍。
  当前阶段**要求用户提供与音频实际播放一致的完整展开歌词**，不做自动
  检测（进阶可选，第二阶段）。如果用户已展开但旧时间戳错位，软先验
  能拉回正确位置
- **重建元数据行而不是原样透传**：元数据可能是 `[ar:]` 这种标准 ID
  标签，也可能是非标准的 `[mood:happy]` 之类。识别后重建容易丢字段；
  按 `raw` 字段原样写回最稳
- **拖拽校准歌词**（来自 001）：技术上能跑但 UX 不达预期（拖拽中 currentTime
  仍前进、视口冻结让人困惑）。已撤回 `3d51a8f`，本阶段就是这条路的替代方案

### 边界情况清单（实现时挨个验）

- 真实歌词的标点 / 空白：对齐时过滤（Unicode `\p{P}`），LRC 输出**保留原样**
- ASR 多字词（"阿里巴巴"作为单个 word 返回）：按时长均分到每个字
- 中英混合：`language_hints: ['zh', 'en']` 覆盖；日韩加 `['ja']` / `['ko']`
- 大小写差异：`charCost` 计 0.1（轻惩罚），不算完全不匹配
- ASR 任务 FAILED：先查 OSS URL 公网可访问 + 音频格式合法
- 5 分钟仍 PENDING：阿里云队列拥堵，提示用户重试而非无限等
- 完全没匹配上（极端情况）：`interpolateNulls` 退化为整体均分
