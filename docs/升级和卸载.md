# windows

## NSIS 安装包 (Setup 版)

用户下载新版 Biu-{version}-win-setup-x64.exe，双击运行。因为你配置了 oneClick: false（非一键静默安装），用户会看到传统的安装向导（让你选“下一步”、“安装目录”）。只要用户选择和旧版本相同的安装目录，NSIS 就会自动覆盖替换旧的核心代码文件，完成升级

---

卸载： 用户在 Windows 的“设置 -> 应用”或“控制面板”中找到 Biu，点击卸载。会弹出一个卸载向导

>
> 注意
> 

卸载软件不会删掉 AppData 里的数据库文件，但你的这份配置里写了 deleteAppDataOnUninstall: true！ 这意味着，当用户通过控制面板卸载你的播放器时，卸载程序会无情地把 AppData/Roaming/Biu 整个文件夹清空！

后果： 用户如果只是想“卸载重装”来解决某个 Bug，他们辛辛苦苦建立的“无上限本地收藏”和“自定义标签”会瞬间灰飞烟灭。

建议： 如果你的播放器未来重度依赖本地 SQLite 数据，强烈建议把这个配置改为 false，或者在卸载时给用户弹一个复选框：“是否保留用户数据？”。

## Portable 便携版

下载新的 Biu-xxx-portable.exe，把旧的那个 .exe 文件直接删掉（或者重命名），双击运行新的即可

---

卸载： 直接把这个 .exe 文件拖进回收站删掉

# mac

双击打开新版的 .dmg 文件，将里面的 Biu.app 图标拖拽到旁边的 Applications (应用程序) 快捷方式上。macOS 会弹窗提示：“名为 Biu 的项目已存在”。用户点击**“替换 (Replace)”**即可

> 
> 注意
>

Gatekeeper 拦截预警 (notarize: false)：
因为配置了未进行苹果公证（notarize: false），当第一次下载你的 DMG 并打开 Biu.app 时，macOS 会弹出一个吓人的警告：“Biu 已损坏，无法打开。你应该将它移到废纸篓”，或者提示“无法验证开发者”。

解决办法： 你需要在发布说明里告诉 Mac 用户，第一次打开时，必须去“系统设置 -> 隐私与安全性”中，往下拉，点击“仍要打开 (Open Anyway)”。之后再覆盖升级，系统就不会再拦截了

---

卸载： 打开“应用程序”文件夹，把 Biu.app 拖进废纸篓
