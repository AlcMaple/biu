# 歌词时间轴对齐 - 纯中文歌测试指南

> 这份文档在 Windows 上独立执行。读这份的人**没有**对话上下文，需要的背景全在下面。

## 背景（5 分钟 ramp up）

- 项目阶段：[002 - 歌词时间轴对齐](ideas/002-歌词时间轴对齐.md)。目标是给「音频 + 不准的 LRC」自动生成时间戳准确的 LRC，作为「精美播放器 → 添加歌词」流程的新入口。
- 算法路径：阿里云 paraformer-v2 异步转写 → 字级时间戳 → DTW 对齐 → 行级 LRC 输出。
- 已通过的测试：王力宏《唯一》（中英混录）端到端跑通，对照见 [biu-windows-test/README.md](../biu-windows-test/README.md)。`wei-yi-aligned.lrc` 在播放器里听已经准了。
- **这次要验证什么**：换一首**纯中文**流行歌，确认在不引入英文/拉丁字符的情况下算法依然稳定。先把纯中文跑通，再回过头处理混录场景，避免变量混在一起难定位问题。
- 测试代码已经在仓库的 [biu-windows-test/test-wei-yi.mjs](../biu-windows-test/test-wei-yi.mjs)——零依赖单文件，包含 parseLrc + DTW + buildLrc + ASR 客户端的完整 snapshot。这次**不改算法**，只换输入。

## 第 1 步：挑歌 + 准备素材【最关键，先做这个】

### 1.1 选一首纯中文流行歌

判断标准：

- **歌词里完全没有英文 / 拉丁字符**（不含 `baby` `oh yeah` 之类）。元数据行 `[ar:]` `[ti:]` 里有英文不算。
- **时长 3-5 分钟**，避免超过 5 分钟超时（脚本里写死了 5 分钟轮询上限）。
- **演唱清晰、伴奏不要太吵**——便于 ASR 识别，便于后续人耳判定对齐准不准。
- 推荐方向：周杰伦慢歌（《晴天》《七里香》《告白气球》）、邓紫棋慢歌、林俊杰慢歌等。**避开**说唱 / 戏腔 / 大量真假音切换的曲子，这些首轮验证不公平。

### 1.3 准备这首歌的 LRC 文本

- 从网易云 / LrcLib / QQ 音乐 / B 站任意一处复制完整 LRC，旧时间戳准不准都无所谓——算法软先验默认 `0.05/秒`，1 秒偏差只多 0.05 分。
- **必须**是 LRC 格式（带 `[mm:ss.xx]` 时间标签），不是纯文本。
- 元数据行（`[ar:]` `[ti:]` `[al:]` 等）保留即可，算法会原样透传。

把内容**手工存成一个文件**，**文件名格式必须是 `<前缀>-bad-lyrics.lrc`**（注意 `bad-lyrics` 三个字必须有，脚本要按这个找）。前缀用歌名拼音，比如 `qingtian-bad-lyrics.lrc`、`gaobaiqiqiu-bad-lyrics.lrc`。放到 `biu-windows-test/` 目录里和 `test-wei-yi.mjs` 同级。

### 判读

- **完整出现 5 个阶段、最后有 `→` 行的对比 → ASR 和算法都跑通了**，进入第 6 步试听。
- 任意阶段失败 → 看下面常见报错表，对照修：

| 报错 | 原因 | 修法 |
|---|---|---|
| `FILE_403_FORBIDDEN` | OSS 签名 URL 过期 | 回第 2 步重新生成 |
| `InvalidApiKey` | DashScope key 错了 | 检查 `.env` 里 `DASHSCOPE_API_KEY` |
| `Cannot find module xxx` | 没 cd 到正确目录 | 确认在 `biu-windows-test/` 下 |
| `--env-file is not a valid argument` | Node < 20.6 | 升级到 22.x |
| `ENOENT: <前缀>-bad-lyrics.lrc` | 第 1.3 步的文件没建好 / 名字不对 | 确认文件名、放在同目录 |
| 5 分钟轮询超时 | 阿里云队列拥堵 / URL 不通 | 等 10 分钟重试，或浏览器直接打开 URL 看能不能下载 |

## 第 6 步：试听验证（核心动作）

拿生成的 `<前缀>-aligned.lrc` 和音频文件用支持 LRC 的播放器配在一起放，凭耳朵判定。

推荐播放器：**VLC**（最稳，免费）。把音频和同名 lrc 放同目录即可自动加载。也可以用 网易云 / Foobar2000 / 千千静听。

听的时候**重点核 3 件事**：

| 检查项 | 通过标准 |
|---|---|
| 整体跟唱 | 每句歌词出现的时刻与人声开口的时刻**相差不超过 200ms**（耳感几乎同步） |
| 前奏不抢跑 | 前奏没结束时，不应该已经出现第一句歌词 |
| 间奏不卡死 | 长间奏期间歌词停在上一句，不会乱跳 |

📝 试听结论（勾选）：

- [ ] **大部分行准** → 算法在纯中文上工作正常 ✅
- [ ] **整体偏早**（歌词先出现，声音才到） → 改脚本顶部 `POST_OFFSET_SEC`，**加大** 0.5 重跑
- [ ] **整体偏晚**（声音先到，歌词才出现） → 改 `POST_OFFSET_SEC`，**减小** 0.5 重跑
- [ ] **个别行明显错** → 记下哪几行错（行号 + 内容），跟前后行的关系
